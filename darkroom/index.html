the API

Your code must define a function:

pixel(x, y) → [r, g, b]   // or [r, g, b, a]

It runs once for every pixel. Channel values are 0–255 (returned values are clamped and rounded for you).

Inside it you can call:

get(x, y) → [r, g, b, a]

get reads the source for the current pass — the original photo on pass one, and the previous pass's output after that — so neighbour lookups never see edits from the pass you're in. Out-of-bounds reads clamp to the nearest edge.

Also in scope: width, height, and everything in Math.

Sliders. Declare a value you want to tweak by eye:

slider(name, default, min, max, step?)

It returns the current value and adds a live slider below the editor — drag it and the photo re-develops. Example:

const amt = slider("amount", 1.5, 0, 4);
function pixel(x, y) {
  const [r, g, b] = get(x, y);
  const l = 0.299*r + 0.587*g + 0.114*b;
  return [l + (r-l)*amt, l + (g-l)*amt, l + (b-l)*amt];
}

Iterations. The × stepper next to Develop runs your function several times in a row, each pass feeding into the next — great for blurs and warps that build up.
