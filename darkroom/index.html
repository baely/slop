Write a pixel(x, y) function, then press Develop.
the API

Your code must define a function:

pixel(x, y) → [r, g, b]   // or [r, g, b, a]

It runs once for every pixel. Channel values are 0–255 (returned values are clamped and rounded for you).

Inside it you can call:

get(x, y) → [r, g, b, a]

get reads the original photo — so neighbour lookups never see your in-progress edits. Out-of-bounds reads clamp to the nearest edge.

Also in scope: width, height. Everything in Math too.
